3. Crypto Selector
- Supported selector order:
  - BTC
  - ETH
  - AVAX
  - DOGE
  - XRP
  - LTC
- BTC selected by default.
- LTC is last and must remain last.
- Clicking a crypto fetches backend data and updates the dashboard.
- If the backend fails, the frontend uses local fallback data.
- Premium feature flag exists:
  - VITE_PREMIUM_FEATURE_ENABLED=true
- When the flag is true, ETH/AVAX/DOGE/XRP/LTC are shown as locked premium assets and open the upgrade modal instead of switching.
- BTC remains free.

4. Manual Refresh Cooldown
- src/hooks/useRefreshCooldown.js provides a 10-second cooldown.
- Manual refresh starts cooldown and disables the refresh button.
- Automatic crypto switching still fetches normally.

5. Frontend Data Flow
CryptoMarketDashboard.jsx:
- selectedCrypto state defaults to BTC.
- theme state initializes from getStoredTheme('dark').
- fetches backend with getMarketOutlook(selectedCrypto).
- uses requestIdRef to avoid stale async responses overwriting newer requests.
- shows loading state.
- shows fallback warning if API fails.
- uses src/data/cryptoDashboardData.js as local fallback.
- renders lazy-loaded PriceHistoryChart and PerformanceComparison with Suspense fallbacks.
